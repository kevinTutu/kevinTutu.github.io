Android端 React-native 更新

react 更新一般为break change 如果要更新react需要和新版react包放入原生app中，否则如果发codepush更新，老用户无法正常收到热更新，收到也会出错。

Android端React的更新

1.替换arr文件：从react native官方demo ,下载react-native init [ProjectName]，从node_modules中的react－native后的android里的aar放入jar（如最新stable版本为0.38.0）

2.替换jnilib库：project/node_modules/react-native/ReactAndroid/src/main/jni/prebuilt
替换armeabi_v7 armeabi x86文件夹下的so文件

3.更改混淆文件：依据demo

4.替换jar包：/Users/xhc/awesomeproject/node_modules/react-native/ReactAndroid中的build.gradle
更改build.gradle的jar包

codePush更新（暂用微软服务器，若后续迁到自己服务器，再做修改）

1.github上找到 react-native-code-push clone工程

2.替换 microsoft.codepush.react 下文件

（注：有坑，了能需要改一部分代码）

React 更新

前端更新库根据packageJson 里的文件版本号确定新版或者旧版
1.npm i react-native@[版本号] [–save] 更新react-native
2.npm i react@[版本号] [–save] 更新react
