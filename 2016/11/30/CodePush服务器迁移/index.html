CodePush 服务迁移

前言：我是一个Android开发人员，从今年6月份开始接触react-native，当时我司已经将react-native集成到项目中了，我算是半路出家的一名小学生。这几个月的时间我也只是写一些component，作为js开发的小白，我的任务可能仅仅是负责将数据渲染到UI组件上，发codepush热更新（也就会几条指令发布，查看等等）。最近呢，我的一位老师跟我说在github上看到了一个code-push-server代码库，我听了之后想了想可以将现在的依赖于微软服务器迁移到自己服务器，这样一方面可以减少对第三方的依赖，另一方面也可以加快热更新的速度，我也就踏上了这条不归路。

特别特别感谢lisong,开源了code-push-server,只有这样的人才能让我们的圈子越来越好！直达链接

我就用小白的语言开始写了，大家见谅

基础工作

你想把微软的server迁移到自己的server,首先你得找台机器。我们是在青云上建了一台ubantu系统机器，通过ssh 进入到这台空白的机器上。

你需要做这几个基础的事儿：(由于我的机器里啥都没有装的东西削微多点)
你要是都有，就别看了：）

  • 装个npm
  • 装个node
  • 装个pm2
  • 装个mysql数据库

安装 - install from npm package

$ npm install code-push-server -g //将code-push-server 命令设置成全局
$ code-push-server-db init –dbhost localhost –dbuser root –dbpassword //初始化mysql数据库
$ code-push-server //启动服务 浏览器中打开 [本机IP]:[3000]

在本地在设置些东西

code-push-cli

$ npm install code-push-cli@latest -g //安装微软本地客户端
$ code-push login [机器iP]:3000 //login in browser account:admin password:123456

之前用过CodePush更新的同学应该知道，命令code-push login就会在浏览器弹出，登录之后就会获得相应的token）

react-native-code-push for react-native

$ npm install react-native-code-push@latest –save -g //安装react-native-code-push

我遇到的一些坑

我是一个小白，所以遇到的坑可能在大神眼里都是小问题，但却浪费我很多时间，记下来给自己提个醒。

1.code-push-sercer config文件找不到。
2.发布时命令总是不对 由于react工程和Android以及iOS完全分离，当使用-p [plist文件路径]时，我看了react-native-code-push文档
文档中Android 需要提供gradle路径，文档上高告诉我也是 -p [gradle文件路径] ,结果就一直报错，后来才查到其实是-g [gradle文件路径]，所以不要相信文档。
